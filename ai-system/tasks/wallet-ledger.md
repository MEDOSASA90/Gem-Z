\## Requirements

\- PostgreSQL database

\- Atomic transactions

\- Idempotency support

\- High security (prevent double spending)

\- Audit logs for every transaction



\## Output Needed

\- Database schema (tables \& relations)

\- API endpoints (deposit, spend, refund)

\- Service logic

\- Edge cases

\- Security risks



\## Constraints

\- No direct balance updates

\- Must be production-ready

\- Must support high concurrency
